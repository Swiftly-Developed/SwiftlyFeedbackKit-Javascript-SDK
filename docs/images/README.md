# FeedbackKit Documentation Images

This directory contains images used in the SDK documentation.

## Required Images

Add the following images to this directory:

| Filename | Dimensions | Description |
|----------|------------|-------------|
| `banner.png` | 1280×640 | Hero banner with FeedbackKit logo and tagline |
| `feedback-list-ios.png` | 390×844 | FeedbackListView on iPhone (with sample data) |
| `feedback-list-mac.png` | 800×600 | FeedbackListView on macOS window |
| `submit-feedback-ios.png` | 390×844 | SubmitFeedbackView on iPhone (form filled) |
| `submit-feedback-mac.png` | 500×400 | SubmitFeedbackView on macOS |
| `feedback-detail-ios.png` | 390×844 | FeedbackDetailView showing votes and comments |
| `theming-example.png` | 800×400 | Side-by-side: default theme vs custom theme |
| `dark-mode.png` | 800×400 | Light mode vs dark mode comparison |

## Optional Images

| Filename | Description |
|----------|-------------|
| `empty-state.png` | Empty feedback list UI |
| `status-badges.png` | Grid showing all status badge colors |
| `quick-start.gif` | Animated demo of integration steps |
| `vote-animation.gif` | Vote button interaction animation |

## Image Guidelines

- **Format:** PNG for static images, GIF for animations
- **Device frames:** Use iPhone 15 Pro and MacBook frames for consistency
- **Sample data:** Use realistic but fictional feedback items
- **Dark mode:** Capture both light and dark variants where applicable
- **Compression:** Optimize images for web (tinypng.com or similar)

## Capturing Screenshots

### iOS Simulator
1. Run the Demo app in iOS Simulator
2. Press ⌘S to capture screenshot
3. Find in ~/Desktop or configured location

### macOS
1. Run the Demo app
2. Press ⌘⇧4, then Space, then click window
3. Find in ~/Desktop

### Device Frames
Use tools like:
- [Rotato](https://rotato.app)
- [MockUPhone](https://mockuphone.com)
- [Facebook Device Mockups](https://design.facebook.com/toolsandresources/devices/)
